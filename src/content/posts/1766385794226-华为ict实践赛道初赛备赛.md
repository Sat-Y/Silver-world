## 2025.11.3
### 整理2023-2024初赛真题
- openGauss采用的是木兰开源协议
- openGauss三权分立
	- 数据库管理
	- 数据安全和数据库审计日志查看与管理的职责被分配给不同的角色和用户
- openGauss数据库逻辑复制支持双主部署、将二进制日志转换为逻辑SQL和表级别复制
- 关于openGauss表的设计
	- 规划好表的结构设计，避免添加字段、修改字段类型或长度
	- 对于频繁更新的astore表，需要制定较小的填充因子
	- 尽量通过聚簇/局部聚簇实现热数据的连续存储，将随机I/O转换为连续I/O
- openGauss数据库主备模式相较与单机的好处
	- 可以进行损坏页自动修复
	- 支持多种故障转移切换和切换模式
	- 可以实现最大可靠性、最大性能和实现最大可用性
	- 最大可能丢失的数据时长（RPO）可以控制，系统恢复正常所需要的最大时长（RTO）可以控制
- 关于openGauss的辅助线程
	- Walwriter负责将已提交事务永久记录写入预写日志文件中
	- Pagewriter用于将脏页数据拷贝到双写区域并落盘
	- Checkpointer用于周期性检查点，将数据脏页刷新到磁盘，确保数据库一致
- 客户端连接工具Data Studio通过JDBC与openGauss数据库进行通信
- gs_checkperf用来让用户了解openGauss的负载情况
- gs_checkperf主要用于检查和展示系统的性能相关信息
- gs_checkos必须使用root用户来执行
- gs_checkos主要用于检查操作系统环境要求、依赖要求、底层配置
- gs_check用于对数据库进行各种检查操作
- gs_ssh用于集群节点间的SSH操作
- gs_collector用于收集数据库的相关信息
- gs_guc工具提供的功能
	- 修改客户端认证策略
	- 修改配置文件中的参数
	- 检查配置文件中的参数
- gs_dump、gs_dumpall用于实现openGauss数据库的逻辑备份
- 关于数据库事务ACID特性
	- A指的是原子性，只具有成功和失败两个选项
	- C指的是一致性，系统状态只有事务前和事务成功后，不存在中间态
	- I指的是Isolation，一事务执行不能被其他事务干扰，个人理解为排他性
	- D指的是持久性，事务执行时机器断电也能恢复到事务成功后的状态
- openEuler默认的SSH服务中必要组成部分为openssh（基础软件包）、openssh-server（用于监听和处理SSH连接请求）、openssh-clients（SSH客户端组件，用于发起请求）
- 属于SSH客户端工具
	- ssh基础的SSH客户端工具，用于建立安全的远程登录会话
	- scp基于SSH协议，用于网路上安全复制文件
	- sftp基于SSH协议，用于安全文件传输
- SSH客户端的配置文件地址/etc/ssh/ssh_config
- 安装openEuler可设置选项
	- 网络和主机名
	- 时间日期
	- 软件选择
- 使用crontab命令可以设置周期定时任务——在openEuler/linux
- job命令主要用于查看当前终端中后台运行的作业——在openEuler/linux
- 在openEuler中使用useradd创建新用户
	- -d命令用于指定新用户家目录
	- -p命令用于指定用户账号和密码
	- -u命令用于指定用户UID（用户标识符）
	- -c命令用于给用户添加备注
- SSD（固态硬盘）性能属于底层硬件需要root用户才可查看
- iptables的表有filter（过滤规则表）、nat（网络地址转换表）、mangle（对数据包进行修改等操作的表）
- 在openEuler中使用&创建后台执行进程
- lscpu可以用于查看cpu型号、数量、频率
- Iscpi用于查看系统硬件信息
- 查看一台linux机器的cpu、swap分区、硬盘等可使用以下命令
	- cat   /proc/cpuinfo
	- cat   /proc/swaps
	- df -lh
- openEuler支持ext4、btrfs、xfs本地文件系统类型
- chgrp命令用于**对文件或目录的所属群组**进行更改
- 使用mkfs -t ext4 /dev/hdb6  可以格式化分区/dev/hdb6

## 2025.11.4
### 整理主体概念
- 什么是openEuler（50%）
	openEuler 是一款由华为主导、社区共建的**开源企业级操作系统**，主要面向服务器、云计算、边缘计算等场景，基于linux内核开发。
- openEuler 的核心特点
	1. **多架构支持**：原生兼容 x86、ARM、RISC-V 等主流芯片架构，能适配不同厂商的硬件产品。
	2. **企业级稳定性**：提供长达 10 年的技术支持周期，满足金融、政务、能源等关键行业对系统稳定性的高要求。
	3. **开源社区驱动**：采用开放的社区治理模式，吸引了包括芯片厂商、整机厂商、软件开发商在内的众多参与者，共同完善生态。
-  主要应用场景
	- **服务器领域**：作为数据库、中间件、大数据平台的运行载体，支撑企业核心业务系统。
	- **云计算领域**：为公有云、私有云环境提供轻量化、高性能的操作系统底座。
	- **边缘计算领域**：针对边缘设备资源有限的特点，提供裁剪优化版本，满足低时延、高可靠的需求。

- 什么是openGauss（30%）
	openGauss 是一款由华为自主研发并开源的**企业级关系型数据库管理系统（RDBMS）**，主要面向金融、政务、能源等对数据高可靠、高性能、高安全有严苛要求的核心业务场景。它以开源社区为依托，致力于构建自主可控的数据库生态。
- openGauss 的核心特点
	1. **高性能**：采用 MPP（大规模并行处理）架构，支持分布式部署，能高效处理海量数据；通过列存引擎、向量计算、自适应查询优化等技术，提升复杂查询和分析型场景的性能。
	2. **高可靠性**：支持主备同步、多副本部署、故障自动切换，确保数据零丢失（RPO=0）和业务快速恢复（RTO 秒级）；提供数据备份、时间点恢复（PITR）等功能，应对硬件故障或人为误操作。
	3. **高安全性**：内置国密算法（如 SM3、SM4），符合等保 2.0 等安全标准；支持数据加密（存储加密、传输加密）、访问控制（细粒度权限管理）、审计日志等特性，保障数据全生命周期安全。
	4. **开源与生态**：采用木兰宽松许可证（Mulan PSL v2）开源，允许商业使用和二次开发；社区吸引了众多企业和开发者参与，适配国产化软硬件（如 openEuler 操作系统、鲲鹏芯片），形成 “软硬一体” 的解决方案。
	5. **易用性**：提供图形化管理工具（如 DAS）、自动化运维脚本，支持在线扩容、平滑升级，降低运维成本；兼容 SQL 标准和主流数据库接口，便于迁移现有业务。
- 主要应用场景
	- **核心业务系统**：如银行交易、政务数据管理、电信计费等，需高可靠和强一致性的场景。
	- **大数据分析**：结合列存和并行计算，支撑海量数据的统计分析、报表生成等。
	- **国产化替代**：作为 Oracle、MySQL 等数据库的替代方案，适配国内软硬件生态，满足自主可控需求。

## 2025.11.11
### 刷题记录
- openEuler中MBR最多支持12个SCSI逻辑分区
- yum remove httpd可以解决依赖关系并卸载系统中已安装的httpd软件
- getfacl -e /data/file1可以查看该文件的所有有效权限
- samba服务再Windows和Linux操作系统下都可以实现文件共享
- 不管是本地软件仓库还是远程软件仓库，仓库中都必须至少有软件包和repodata目录
- 在Linux系统中，groupmod -g可以修改用户组的id
- gpasswd -M user01，user02，user03 admin  批量将admin组的成员设置名称
- 在openEuler系统中，扩展分区占用一个主分区的名额
- 一次性定时任务at命令
- bash是默认shell解析用户输入的命令
- Linux系统中根据用户uid来区分用户类型
- rpm -qi可以列出rpm软件的描述信息
- rpm -ql可以查看软件是否安装
- chown可以同时**修改文件的拥有者和所属组**
- openEuler系统中Su-在切换用户时，用户身份和shell环境一起切换
